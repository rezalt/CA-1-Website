Observer pattern used for the client (server listener).
As soon as the client logs in she/he creates a new thread which begins to listen for incoming messages (from the server). The way the listener works is just that it calls the client.receive() method which blocks until a new message is received. When a new message arrives it notifies the GUI (Observer) and makes the necessary changes on the GUI ie. update the message field or the list of users online. In this way since client.receive() is a blocking call the GUI isn’t locked since the listener runs on a different thread than the GUI.
